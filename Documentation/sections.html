Sections

A section represents a part of an web page main content.

Introduction

The main content of the page (located within the main or div tag with a class page-main) is a combination of multiple blocks called Section.

Parameters

A section appearance changes when certain classes is added to it.

Regular parameters

hero-a class

This class is used when a section represents a web page header (called hero). Then its content appearance is set via CSS by defining rules for .hero-a{ ... } class.

section-a class

This class is used by other type of a section (different than the hero). Its content appearance is set via CSS by defining rules for .section-a{ ... } class.

fullscreen-h class

This make the section to have at least a 100% viewport height.

Use class fullscreen-h-lg to apply this option only for large devices.

content-white class

This class tells that the section content will have a brigh text color (white, yellow, ...). Then, when user scroll into this section, HTML tags with class change-on-section-colorwill get also a content-white class.

Check out the home-3.html file to see how to use it.

content-dark class

This class tells that the section content will have a dark text color (black, blue, ...). Then, when user scroll into this section, HTML tags with class change-on-section-colorwill get also a content-dark class.

attribute id="SECTION_ID"

Use this attribute when you use the scroll navigation option on the main page. It will be the id of the section.

Fullscreen slideshow parameters

These parameters are used when you use the slideshow scroll with fullpage js option on the main page.

Check out the index-fullpage.html file to see how to use them.

fp-auto-height-responsive class

Use this class when you use the fullpage.js plugin. It will make the height af the section to fit the content on mobile devices.

attribute data-section="SECTION_REF"

Use this attribute when you use the fullpage.js plugin. It will become the id of the section.
